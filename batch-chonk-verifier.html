TL;DR

Problem: Each IVC proof verification requires an expensive IPA multi-scalar multiplication (MSM) over the Grumpkin SRS. At ~200ms per proof, verifying 30 P2P transactions sequentially takes 6 seconds.

Solution: IPA::batch_reduce_verify checks N proofs in a single MSM. Cost is O(1) MSMs for valid batches, O(k log N) for k bad proofs via bisection. Reduce phase (sumcheck, databus, Goblin) runs in parallel across worker threads.

Result: ~30x throughput improvement for honest batches. Bad proofs are isolated without re-running the expensive reduce phase.

Why Batch?

Chonk IVC verification has two phases with very different cost profiles:

Phase 1: Reduce per-proof

MegaZK sumcheck, databus, Goblin merge/ECCVM/Translator. Produces an IPA claim + transcript. Embarrassingly parallel across proofs.

Phase 2: IPA Check per-batch

Single IPA::batch_reduce_verify over all claims. One MSM regardless of batch size. This is the key saving.

Without batching, each proof pays the full MSM cost. With batching, N proofs share one MSM. The reduce phase parallelizes naturally via work-stealing.

The Algorithm

Coordinator Loop

loop {
    // Wait for proofs (no timeout - process immediately when idle)
    batch = take_up_to(batch_size, queue)

    // Phase 1: Parallel reduce (work-stealing, N workers)
    results = parallel_reduce(batch)   // IPA claims, not yet checked
    emit failures from reduce phase

    // Phase 2: Single batch IPA check
    ok = IPA::batch_reduce_verify(claims, transcripts)

    if ok:  emit_ok(all)              // Happy path: 1 MSM total
    else:   bisect(indices, depth=0)   // Find the bad ones
}

Bisection

bisect(indices, depth):
    if |indices| == 1:
        emit FAILED                      // Base case
        return

    left, right = split(indices)
    left_ok = batch_check(left)

    if left_ok:
        emit_ok(left)                       // Left is clean
        bisect(right, depth+1)               // All bad must be right
    else:
        bisect(left, depth+1)                // Left has bad proofs
        right_ok = batch_check(right)
        if right_ok: emit_ok(right)
        else: bisect(right, depth+1)          // Both halves bad

Key optimization

When the left half passes, we skip batch_check on the right and recurse directly. Since IPA batch verification has negligible soundness error, a passing left half means all failures are in the right. This saves one MSM per level where left succeeds.

Soundness Argument

Claim: Every proof is correctly classified as OK or FAILED.

1. batch_reduce_verify is sound

IPA::batch_reduce_verify takes a random linear combination of N IPA opening claims. The probability of a false positive (invalid claim passing) is negligible in the field size (~2-254 for Grumpkin). This is a standard Schwartz-Zippel argument.

2. If left passes, all failures are in right

If batch_check(left) returns true, each claim in the left subset individually satisfies IPA verification (with overwhelming probability). Therefore, the claim(s) causing the full-batch failure must be in the right subset. Skipping right's re-check is sound.

3. Completeness

Every index appears in exactly one of {left, right} at each level. Recursion terminates at |indices|=1. Every proof is either:

  • Emitted as OK via emit_ok at some bisection level, or
  • Identified as FAILED at the base case (single proof fails batch_check)

No proof is dropped or double-counted.

4. Reduce phase is not repeated

The expensive reduce_to_ipa_claim (sumcheck, Goblin, etc.) runs once in Phase 1. Bisection only re-runs the cheap batch_check (IPA MSM) on subsets. The IPA claims and transcripts from Phase 1 are reused.

C++ Implementation

chonk_batch_verifier.cpp — coordinator_loop (the heart) 
void ChonkBatchVerifier::coordinator_loop() {
    while (true) {
        // Collect batch under lock
        std::vector<VerifyRequest> batch;
        {
            std::unique_lock lock(mutex_);
            cv_.wait(lock, [this] { return shutdown_ || !queue_.empty(); });
            size_t take = std::min(queue_.size(), (size_t)batch_size_);
            batch = move(queue_[0..take]);
            // Filter invalid vk_index → emit FAILED immediately
        }

        // Phase 1: parallel reduce (work-stealing)
        auto results = parallel_reduce(batch);
        // Emit reduce-phase failures, collect passed indices

        // Phase 2: batch IPA
        bool ok = batch_check(results, passed);

        if (ok) emit_ok(results, passed, ...);
        else    bisect(results, passed, 0, ...);
    }
}
parallel_reduce — work-stealing across N workers 
std::atomic<size_t> work_index{ 0 };
for (w = 0; w < num_workers; w++) {
    workers.emplace_back([&]() {
        set_parallel_for_concurrency(1);  // single-threaded per worker
        while (true) {
            idx = work_index.fetch_add(1);
            if (idx >= num_proofs) break;
            try {
                ChonkNativeVerifier v(vks_[req.vk_index]);
                results[idx] = v.reduce_to_ipa_claim(req.proof);
            } catch (...) { results[idx] = FAILED; }
        }
    });
}
batch_check — single MSM over all IPA claims 
bool batch_check(results, indices) {
    set_parallel_for_concurrency(num_cores_);  // multi-threaded MSM
    // Collect claims + transcripts
    auto ipa_vk = VerifierCommitmentKey<Grumpkin>{ECCVM_FIXED_SIZE};
    return IPA<Grumpkin>::batch_reduce_verify(ipa_vk, claims, transcripts);
}
bisect — binary search for bad proofs 
void bisect(results, indices, depth, reduce_start) {
    if (indices.size() == 1) {
        on_result_(VerifyResult::failed(...));  // base case
        return;
    }
    left, right = split(indices);
    bool left_ok = batch_check(results, left);

    if (left_ok) {
        emit_ok(left);
        bisect(right, depth+1);  // skip right check
    } else {
        bisect(left, depth+1);
        if (batch_check(right)) emit_ok(right);
        else bisect(right, depth+1);
    }
}
